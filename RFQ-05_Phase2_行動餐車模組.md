# Easy2Eat RFQ 05：Phase 2 行動餐車模組開發

文件版本：v1.0  
建立日期：2026-03-13  
專案名稱：Easy2Eat 北部小農市集整合平台

## 1. 專案目標

在既有小農平台成功上線後，擴充行動餐車模組，提供餐車位置、菜單、預訂、通知與後台管理能力。

## 2. 專案背景

第二階段目標為延伸既有平台的服務範圍，將「農夫」平行擴充為「餐車業者」。本模組應以既有會員、商品、訂單、推播與地圖能力為基礎重用，不應另起一套獨立系統，以降低維運成本與資料分裂風險。

## 3. 開發原則

- 應以既有後端與會員系統為基礎擴充。
- 應盡量重用商品、訂單、通知、地圖等既有模組。
- 不建議另起一套獨立系統。

## 4. 開發範圍

- 餐車業者資料管理
- 每日出車位置管理
- 即時或準即時位置更新
- 餐車地圖
- 菜單管理
- 線上預訂
- 取餐通知
- 收藏與推播
- 餐車後台

## 5. 功能需求

### 5.1 餐車業者資料

- 建立餐車品牌資料。
- 管理介紹、圖片、營業資訊。
- 設定營業區域或常見停靠點。

### 5.2 出車位置管理

- 每日登記出車位置。
- 設定開始與結束時間。
- 提供 1-click 打卡更新位置。
- 平台管理員可代為修改。

### 5.3 餐車地圖

- 顯示附近餐車。
- 顯示時間區間。
- 點擊查看餐車摘要與詳情。

### 5.4 菜單與預訂

- 菜單建立與管理。
- 餐點圖片與說明。
- 設定可預訂時段。
- 預約取餐時間。
- 線上付款或到場付款。

### 5.5 餐車後台

- 管理每日位置。
- 管理菜單。
- 查看訂單。
- 更新備餐狀態。
- 發送或觸發通知事件。

### 5.6 消費者功能

- 收藏餐車。
- 接收出車通知。
- 查看歷史消費紀錄。
- 查詢預訂狀態。

## 6. 技術與架構要求

- 重用既有帳號與訂單系統。
- 地圖服務重用既有 Google Maps 能力。
- 推播重用既有 FCM。
- 即時性可先採 polling 或輕量 WebSocket，不要求一開始即做高精度背景追蹤。

## 7. 建議技術棧

- 後端：沿用既有 `NestJS` 或 `Next.js` 後端
- Web / 後台：沿用既有 `Next.js`
- App：沿用既有 `Flutter`
- 即時資料：`Socket.IO` 或 polling
- 推播：`Firebase Cloud Messaging`

## 8. 交付物

- 後端擴充 API
- Web 與後台擴充頁面
- App 端新增頁面
- 資料模型與 migration
- 測試文件
- 上線交接文件

## 9. 驗收條件

- 餐車業者可建立與更新位置。
- 消費者可在地圖查看餐車。
- 可完成預訂與查詢。
- 可收到餐車出車通知。
- 平台可在後台查詢餐車與訂單資料。

## 10. 排除項目

- 高頻即時 GPS 追蹤
- 外送配送系統
- 動態定價
- 多城市深度營運管理
- 第三方外送平台整合

## 11. 廠商回覆格式

請廠商至少回覆以下內容：

- 如何重用既有模組
- 餐車模組預估工期
- 即時位置技術建議
- 報價與人力配置
